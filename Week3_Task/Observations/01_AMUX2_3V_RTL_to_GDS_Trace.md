# AMUX2_3V RTL-to-GDS Trace Investigation

## Objective

One of the primary objectives of Week 3 was to determine whether the analog hard macro `AMUX2_3V` successfully propagates through the complete mixed-signal physical design flow. During the Week 2 investigation, the absence of a standalone `AMUX2_3V.gds` file and the difficulty in identifying the macro within the final layout raised the question of whether the macro was being removed during implementation.

To answer this question, the design was traced stage-by-stage from RTL through synthesis, floorplanning, placement, routing, DEF generation and finally the generated GDS.

---

# Investigation Procedure

The following implementation stages were inspected.

```
RTL
    ↓
Yosys Synthesis
    ↓
OpenROAD Floorplan
    ↓
Global Placement
    ↓
Detailed Routing
    ↓
Final DEF
    ↓
Generated GDS
```

At every stage, the generated design netlist or physical database was searched for the `AMUX2_3V` instance.

---

# RTL Verification

The original RTL was searched for the macro instance.

Result:

```
design_mux.v:
AMUX2_3V AMUX2_3V (
```

This confirms that the analog macro is instantiated in the original RTL design.

---

# Yosys Synthesis Verification

The synthesized netlist generated by Yosys was inspected.

Result:

```

AMUX2_3V AMUX2_3V (
```

Observation:

The analog macro was preserved during synthesis and was not optimized away.

---

# DEF Verification

The routed DEF file was searched for the macro instance.

Result:

```
AMUX2_3V AMUX2_3V + PLACED (112240 76160) N;
```

Additional routed nets connected to the macro were also identified.

```
AMUX2_3V I0
AMUX2_3V I1
AMUX2_3V out
AMUX2_3V select
```

Observation:

The DEF confirms that OpenROAD physically placed the analog macro and successfully routed all required signal connections.

---

# LEF Verification

The LEF abstraction of the macro was inspected.

Result:

```
SIZE 8.740 BY 5.440 ;
```

The placement coordinates reported in the DEF correspond to a macro of the expected dimensions.

---

# GDS Inspection

The final streamed-out GDS was opened in KLayout.

The region corresponding to the DEF placement coordinates was inspected visually.

A rectangular macro-sized region was observed with the expected signal pin labels including:

* select
* out

The observed geometry matches the placement information reported in the DEF database.

---

# Findings

The stage-by-stage investigation demonstrates that the AMUX2_3V analog macro is preserved throughout the complete implementation flow.

| Stage            | Status                    |
| ---------------- | ------------------------- |
| RTL              | ✓ Present                 |
| Yosys Synthesis  | ✓ Present                 |
| Floorplan        | ✓ Present                 |
| Global Placement | ✓ Present                 |
| Detailed Routing | ✓ Present                 |
| DEF Database     | ✓ Placed                  |
| GDS Layout       | ✓ Macro region identified |

---

# Conclusion

The investigation shows that the AMUX2_3V macro is preserved throughout the complete implementation flow. The macro is present in the RTL, synthesized netlists, placement database, routed DEF, and corresponding region of the final GDS layout. Although no standalone AMUX2_3V.gds file or EXTRA_GDS_FILES configuration was found in the reproduced flow, the final layout still contains the placed macro. This indicates that the reproduced implementation successfully incorporates the analog macro without requiring an explicitly supplied macro GDS file in the project configuration. This investigation corrected the initial assumption made during Week 2 that the analog macro might have been absent from the final design and provided a complete trace of the macro throughout the implementation process.
